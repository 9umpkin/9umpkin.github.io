239. 滑动窗口最大值

给你一个整数数组 nums，有一个大小为 k 的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 k 个数字。滑动窗口每次只向右移动一位。

返回 滑动窗口中的最大值 。

示例 1：

示例 2：

提示：

  • 1 <= nums.length <= 105
  • -104 <= nums[i] <= 104
  • 1 <= k <= nums.length

方案一

整体思路就是使用一个队列来维护窗口，右边界入队，左边界出队，同时记录每一时期队列的一个最大值。

最大值是比较难处理的，不能使用一个max来进行简单的判断。涉及到left\right\max三个值，要讨论的情况太多了。

代码随想录推荐了一个单调队列。

这个单调队列的思想就是，所有进队的值都要进行排序，那么第一个出队的值就是最大值。

但是这样怎么处理窗口的移动呢？

实际上，我们无需把所有的值都入队，只需要把可能成为最大值的元素入队即可，

其实队列没有必要维护窗口里的所有元素，只需要维护有可能成为窗口里最大值的元素就可以了，同时保证队列里的元素数值是由大到小的。

什么叫可能成为窗口里的最大值：其后面的一个元素不比他大，那么我们就认为他有可能是最大值

设计单调队列的时候，pop，和push操作要保持如下规则：

  1. pop(value)：如果窗口移除的元素value等于单调队列的出口元素，那么队列弹出元素，否则不用任何操作
  2. push(value)：如果push的元素value大于入口元素的数值，那么就将队列入口的元素弹出，直到push元素的数值小于等于队列入口元素的数值为止

方案二

采用Deque的效率比较低，也可以采用一个基础的LinkedList来自己完成这个单调队列。

每一个循环要做几件事情：

  • 找到窗口中的最大值并记录下来
  • 移除nums[i - k]
  • 加入nums[i]
  • 保持窗口降序排列

347. 前 K 个高频元素

给你一个整数数组 nums 和一个整数 k ，请你返回其中出现频率前 k 高的元素。你可以按 任意顺序 返回答案。

示例 1:

示例 2:

提示：

  • 1 <= nums.length <= 105
  • k 的取值范围是 [1, 数组中不相同的元素的个数]
  • 题目数据保证答案唯一，换句话说，数组中前 k 个高频元素的集合是唯一的

进阶：你所设计算法的时间复杂度 必须 优于 O(n log n) ，其中 n 是数组大小。

方案一

最基本的做法就是先统计再排序，但是这个思路太常规了，这里先不讨论。主要思考从栈和队列入手操作。

  1. 采用一个哈希表来记录元素出现的频率
  2. 使用优先级队列存储元素，保证队列按照元素的频率降序排列

什么是优先级队列呢？

其实就是一个披着队列外衣的堆，因为优先级队列对外接口只是从队头取元素，从队尾添加元素，再无其他取元素的方式，看起来就是一个队列。优先队列有另一个名字：二叉堆

什么是堆呢？

堆是一棵完全二叉树，树中每个结点的值都不小于（或不大于）其左右孩子的值。 如果父亲结点是大于等于左右孩子就是大顶堆，小于等于左右孩子就是小顶堆。

优先队列和单调队列的区别：

  • 单调队列：如果有比你更年轻的人来，那么你就要被踢出去咯。
  • 优先队列：优先级高的站前面，优先级低的站后面。

“前K个最大/最小/最频繁”等等一系列的题目，首先就要想到堆排序。这里有一些基础知识点的补充：

  • 优先队列实际上就是二叉堆，排序的效率为O(nlogn)

  • Java有内置的优先队列PriorityQueue，可以传入自定义比较器

  • import java.util.PriorityQueue;
import java.util.Comparator;
class Main {
    public static void main(String[] args) {

        //创建优先级队列
        PriorityQueue<Integer> numbers = new PriorityQueue<>(new CustomComparator());
        numbers.add(4);
        numbers.add(2);
        numbers.add(1);
        numbers.add(3);
        System.out.print("PriorityQueue: " + numbers);
    }
}

class CustomComparator implements Comparator<Integer> {

    @Override
    public int compare(Integer number1, Integer number2) {
        int value =  number1.compareTo(number2);
        //元素以相反的顺序排序
        if (value > 0) {
            return -1;
        }
        else if (value < 0) {
            return 1;
        }
        else {
            return 0;
        }
    }
}
